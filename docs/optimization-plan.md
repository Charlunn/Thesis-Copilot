# QNU Thesis Copilot 优化方案

## 1. 当前状态

### 1.1 MVP 已完整实现

当前 MVP 已经具备完整的本地闭环：

- ✅ 项目工作区与 `state.json` 持久化
- ✅ 推荐文献 JSON 导入与 PDF 规范化处理
- ✅ BibTeX 导入与关联
- ✅ 大纲导入与树编辑
- ✅ 切块规划与正文逐块生成工作台
- ✅ 上下文压缩链路
- ✅ **本地 `.docx` 导出（含青海师范大学正式模板注入）** ← 新完成

### 1.2 正式模板已接入

- ✅ 模板文件 `qnu-undergraduate-v1.docx` 已放入 `backend/assets/templates/`
- ✅ 导出时优先使用正式模板
- ✅ 模板不存在时回退到基础样式导出

### 1.3 下一步

现阶段可以支撑"小范围稳定使用"，距离"校内广泛推广"和"正式比赛答辩级展示"还有若干优化空间。

## 2. 优化目标

优化工作的核心目标不是盲目加功能，而是围绕三件事展开：

1. 提高成功率
2. 降低学习成本
3. 提升最终文档质量与可维护性

## 3. 优先级优化项

### P0：直接影响交付质量

- [ ] 完善参考文献格式化逻辑，进一步逼近 GB/T 7714 标准输出
- [ ] 为块生成结果增加"缺失引用 / 异常占位符 / 空块"检查
- [ ] 为导出阶段增加结构完整性校验，例如一级标题缺失、章节顺序异常、引用编号越界

### P1：直接影响使用体验

- [ ] 把所有 Prompt 模板收敛到 `backend/assets/prompts/` 统一版本管理
- [ ] 在桌面端为每一步增加更明确的"去哪一个模型、粘贴什么、期待返回什么"教学说明
- [ ] 增加项目恢复横幅，明确告诉用户"当前停留在哪一步、下一步应该做什么"
- [ ] 增加导出历史列表，支持一键打开最近几次导出结果

### P2：直接影响稳定性

- [ ] 为 `state.json` 增加版本迁移器，避免后续字段演进导致旧项目无法打开
- [ ] 增加自动备份机制，在关键节点写入 `state.backup.json`
- [ ] 为 PDF 和导出文件增加更明确的异常日志
- [ ] 对桌面端增加更多集成测试，覆盖多项目切换和阶段恢复

### P3：为校内推广做准备

- [ ] 增加 Windows 安装包和 sidecar 打包脚本
- [ ] 增加固定模板资产检查，避免用户机器缺模板时不知所措
- [ ] 增加"项目分享包"导出能力，便于同学之间迁移工作区
- [ ] 增加 UI 级错误说明，将后端错误翻译成更友好的中文提示

## 4. 技术优化建议

### 4.1 后端

- 引入独立的引用服务，将"引用占位符还原、文献排序、参考文献格式化"从导出服务中拆出
- 为 BibTeX 解析补充更稳健的字段提取器，减少对简单正则的依赖
- 将导出日志结构化，记录块数、文献数、模板路径、耗时与错误原因

### 4.2 前端

- 将 BibTeX 导入、导出页拆成独立组件，降低 `App.tsx` 复杂度
- 引入阶段导航条，让用户能从顶部直接看到完整进度
- 对长路径、长 Prompt 和长 JSON 增加折叠展示与复制反馈

### 4.3 测试

- 后端增加导出内容级测试，而不仅仅是"文件存在"测试
- 桌面端补充端到端流程测试，覆盖新建项目到导出完成的完整链路
- 对异常输入增加回归测试，例如错误 BibTeX、错误阶段导出、导出时缺块

## 5. 已完成里程碑

- ✅ 正式模板接入（`qnu-undergraduate-v1.docx`）
- ✅ 导出质量基本可用
- ✅ Prompt 资产化（7 个模板统一管理）
- ✅ 项目备份与恢复
- ✅ 导出历史记录

## 6. 推荐实施顺序

### 第一阶段（P0 质量提升）

- 完善 GB/T 7714 参考文献格式化
- 块生成质量校验
- 导出结构完整性检查

### 第二阶段（P1 体验增强）

- 桌面端教学增强
- 阶段进度导航条
- 错误提示优化

### 第三阶段（P2 稳定性）

- 状态迁移与备份
- 更多集成测试
- 异常日志优化

### 第四阶段（P3 推广准备）

- 安装包与校内试用分发
- 项目分享包
- 更完整的用户文档

## 7. 结论

这个项目下一步最值得投入的方向，不是继续堆更多"AI 能力"，而是把已经验证有效的链路打磨到更稳定、更清晰、更符合学校模板要求。只要成功率和导出质量继续提升，它就具备很强的校内传播能力。

**当前状态**：MVP 完整闭环已实现，青海师范大学正式模板已接入，具备基本使用价值。
