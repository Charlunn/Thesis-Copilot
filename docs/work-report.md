# QNU Thesis Copilot 述职报告

## 一、项目概述

QNU Thesis Copilot 是一个面向青海师范大学本科毕业论文场景的半自动桌面工具。项目目标不是做开放式聊天产品，而是把已经验证有效的人机协作流程固定下来，帮助学生以更高成功率完成论文初稿生成和排版导出。

## 二、本人承担的主要工作

在本阶段中，我主要负责了以下工作：

- 设计并实现本地优先的桌面端技术路线
- 建立 Python 本地内核与 FastAPI 适配层（14 个服务模块）
- 完成项目工作区、`state.json`、PDF 处理、JSON 校验等基础能力
- 完成大纲导入、切块规划、正文块生成工作台等核心流程
- 补齐 BibTeX 导入和 `.docx` 导出闭环
- **接入青海师范大学正式模板 `qnu-undergraduate-v1.docx`**
- 完成前端 7 个核心组件开发
- 编写测试、完善文档，并将项目整理成可解释、可维护的 MVP 仓库

## 五、技术难点与解决方式

### 5.1 流程复杂，阶段多

论文生成流程天然跨越文献、结构、正文、导出多个阶段。解决方式是以 `state.json` 作为唯一事实源，并通过明确的工作流阶段来约束操作顺序。

### 5.2 AI 输出不稳定

AI 常出现 Markdown 包裹、解释性文字、尾部截断等问题。解决方式是建立统一 JSON 清洗与合同校验能力，让所有阶段输出都经过同一套解析入口。

### 5.3 桌面端与本地后端协作

Electron 与 Python 属于双技术栈协作。解决方式是让 FastAPI 仅承担薄适配层，把真正业务逻辑都沉到 Python 服务层，保证结构清晰。

### 5.4 导出闭环与模板接入

此前导出仍是基础样式回退。解决方式是：
1. 新增 BibTeX 管理和本地 `.docx` 导出服务
2. **接入青海师范大学正式模板文件**
3. 完成模板注入机制
4. 先完成可运行闭环，再为更多模板接入保留扩展空间
